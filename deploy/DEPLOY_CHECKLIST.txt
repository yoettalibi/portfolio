DEPLOY CHECKLIST — ettalibi
============================

DIRECTORY STRUCTURE ON THE SERVER
----------------------------------
/home/<user>/
  backend/            <- upload the entire Laravel project here
  public_html/        <- web root (your domain points here)
    api/
      .htaccess       <- from deploy/api/.htaccess
      index.php       <- from deploy/api/index.php
    index.html        <- from frontend/dist/  (all dist/ contents)
    assets/
    .htaccess         <- from frontend/dist/.htaccess  (auto-generated by Vite build)
    ... (rest of dist/)


STEP 1 — Build the frontend
---------------------------
  cd frontend
  npm run build
  -> Output is in frontend/dist/
  -> Upload ALL files in dist/ into public_html/


STEP 2 — Upload the backend
----------------------------
  Upload the entire backend/ folder to /home/<user>/backend/
  Do NOT upload it inside public_html/.
  The vendor/ folder must be present (upload it or run composer install on server).


STEP 3 — Upload the API bridge
--------------------------------
  Upload deploy/api/.htaccess  -> public_html/api/.htaccess
  Upload deploy/api/index.php  -> public_html/api/index.php


STEP 4 — Configure the backend .env
-------------------------------------
  Copy deploy/backend.env.production to /home/<user>/backend/.env
  Fill in real values:
    APP_URL            = https://yourdomain.com
    FRONTEND_URL       = https://yourdomain.com
    DB_DATABASE        = (cPanel database name)
    DB_USERNAME        = (cPanel database user)
    DB_PASSWORD        = (cPanel database password)
    RESEND_API_KEY     = (your real Resend key)
    MAIL_FROM_ADDRESS  = contact@yourdomain.com
    SESSION_DOMAIN     = .yourdomain.com

  Generate a fresh app key:
    php artisan key:generate
  (or run it via cPanel terminal / SSH)


STEP 5 — Run database migrations
----------------------------------
  Via SSH or cPanel terminal:
    cd /home/<user>/backend
    php artisan migrate --force
    php artisan db:seed --class=AdminSeeder


STEP 6 — Set folder permissions
---------------------------------
  backend/storage/           -> 755 (writable by PHP)
  backend/storage/logs/      -> 755
  backend/bootstrap/cache/   -> 755


STEP 7 — Clear Laravel caches
-------------------------------
  php artisan config:cache
  php artisan route:cache
  php artisan view:cache


VERIFY
------
  https://yourdomain.com              -> React SPA loads
  https://yourdomain.com/about        -> React Router page (not 404)
  https://yourdomain.com/api/up       -> {"status":"up"}  (Laravel health check)
  https://yourdomain.com/api/settings -> JSON response from Laravel


NOTES
-----
- The deploy/api/index.php bridge adjusts SCRIPT_NAME before booting Laravel
  so Symfony routes requests correctly (see comment in that file).
- CORS is not required because frontend and API share the same origin.
- SESSION_DRIVER=file is safer on shared hosting than database.
- Never expose backend/ through public_html — keep it one level above.
